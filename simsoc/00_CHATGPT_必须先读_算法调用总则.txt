三、算法调用入口
------------------------------------------------------------

当前算法调用入口以 maintask.c 中的 maintask() 为准。

函数形式：

void maintask(int voltage, int current, int cap, int cellnum, int temp, int init_voltage)

输入参数类型已经写在代码里，这里只记录逻辑含义和调用规则。

maintask() 是 100ms 周期调用函数。
外部输入数据必须按 100ms 周期送入。
每调用一次 maintask()，代表算法运行一个 100ms 周期。


四、maintask 输入参数规则
------------------------------------------------------------

1. voltage
   当前电池总电压输入。
   单位按 maintask.c 注释执行。
   数据源进入算法前必须已经符合 maintask 要求的单位。
   maintask 内部不负责理解外部数据来源。

2. current
   当前电流输入。
   单位按 maintask.c 注释执行。
   数据源进入算法前必须已经符合 maintask 要求的单位。
   maintask 内部不负责理解外部数据来源。

3. cap
   电池容量参数。
   用于 soc_reset() 初始化容量相关变量。
   调用者必须按当前算法约定传入。

4. cellnum
   电池节数。
   铅酸单节按 12V 理解。
   第一次运行初始化时需要电池节数。
   如果明确知道电池节数，应直接传入明确 cellnum。

5. temp
   温度输入。
   用于算法内部温度相关逻辑。
   如果测试数据没有温度变化，可按约定传固定温度。

6. init_voltage
   初始电压。
   第一次运行初始化时用于 OCV 初始化初始 SOC。
   也就是 tick == 0 时使用。
   初始化后，后续 100ms 周期主要依靠持续输入 voltage/current/temp 等数据运行。


五、初始化逻辑
------------------------------------------------------------

1. maintask 内部通过 tick 判断是否第一次运行。

2. 当 tick == 0 时：
   - 进行算法初始化。
   - 使用 cellnum 确定电池节数。
   - 使用 init_voltage 进行 OCV 初始化初始 SOC。
   - 初始化完成后，算法进入后续周期运行。

3. 第一次调用必须提供有效初始化信息：
   - 电池节数 cellnum。
   - 初始电压 init_voltage。
   - 初始 voltage/current/temp/cap。

4. 后续调用：
   - 按 100ms 周期不断输入新的 voltage/current/temp 等数据。
   - 不再重复进行第一次初始化。

5. 如果要重新开始一组新的测试：
   - 应保证算法状态重新初始化。
   - 可以通过重新启动程序、重新加载算法实例、或按用户确认的方式重置状态。
   - 不要在未确认的情况下擅自改代码增加 reset 接口。


六、周期输入规则
------------------------------------------------------------

1. maintask 是 100ms 周期。

2. 外部数据必须整理为 100ms 周期后再输入算法。

3. 每个 100ms 点调用一次：

   maintask(voltage, current, cap, cellnum, temp, init_voltage)

4. 第一次调用用于初始化。

5. 后续每个 100ms 调用一次，持续输入数据。

6. 数据如何从原始文件转换成 100ms，属于 data 目录的数据处理问题，不记录在 simsoc/readme.log 中。
   simsoc 只要求最终输入 maintask 的数据已经满足：
   - 单位正确。
   - 周期正确。
   - 初始化参数正确。


七、输出观测规则

2. 观测什么量由用户每次测试时指定。

3. 如果用户没有重新指定输出量，可以参考上一次的输出内容。

4. 常见可能观测量包括但不限于：

八、调用方式总结
------------------------------------------------------------

最小正确调用逻辑：

1. 准备好已经符合 maintask 输入要求的数据。
2. 数据周期整理为 100ms。
3. 第一次调用时提供：
   - voltage
   - current
   - cap
   - cellnum
   - temp
   - init_voltage
4. 第一次调用 tick == 0，算法初始化。
5. 后续每 100ms 继续调用 maintask，输入新的 voltage/current/temp 等数据。
6. 每次调用后，根据测试需求读取并输出用户想看的变量。

伪代码：

for each 100ms sample:
    if first_sample:
        maintask(voltage, current, cap, cellnum, temp, init_voltage)
    else:
        maintask(voltage, current, cap, cellnum, temp, init_voltage_or_0)

    output selected variables

注意：
- 第一次调用必须保证初始化参数有效。
- 后续调用重点是 100ms 周期连续输入。
- 不要把数据处理逻辑混进算法目录说明。

十二、修正：初始化参数必须包含容量 cap（追加 2026-07-11）
------------------------------------------------------------

前面“防遗忘调用检查清单”里强调了 cellnum 和 init_voltage，但还需要明确：
第一次初始化时也必须有有效 cap。

原因：
- cap 是电池容量参数。
- cap 会参与 soc_reset() 初始化容量相关变量。
- 如果 cap 不正确，即使 cellnum 和 init_voltage 正确，容量、积分、SOC 变化幅度等都会不可靠。

因此第一次 tick == 0 初始化时，必须同时保证：
- cap 正确
- cellnum 正确
- init_voltage 正确
- 初始 voltage/current/temp 有效

后续执行算法前的检查应改为：
- 不要忘记第一次 tick == 0 初始化。
- 不要忘记 maintask 是 100ms 周期。
- 不要忘记初始化需要 cap、cellnum 和 init_voltage。

最小初始化调用仍然是：
maintask(voltage, current, cap, cellnum, temp, init_voltage)

其中 cap/cellnum/init_voltage 都是初始化关键参数。
